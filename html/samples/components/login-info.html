Login

Handle your User's Login
Use the attribute tm1-after-login to call your custom Javascript functions after the user has logged in.

Checkout this page's controller - js\controllers\samples\components\login-session.js on how the above function and the $ngBootbox service was utilize to show the pop-up.

Multiple instances will automatically be shown in a dropdown. If there is only one, the dropdown is hidden.
Customize your Login Page

Customize the look and feel of your login via tm1-ui-class

Choose "Custom" from the above to see one of the many possibilities!

.custom-login * input[type]{
        font-size: 20px; 
        border: none;
        border-bottom: 2px solid rgb(255, 43, 85);
        border-radius: 0px;
      }
      
      .custom-login * .panel-heading{
        background-color: rgb(255, 43, 85);
        color: White;
      }
      
      .custom-login > .panel{
        border-color: rgb(255, 43, 85);
      }
      
      .custom-login * button{
        background-color: rgb(255, 43, 90);
        border-color: White;
      }
      
      .custom-login * button:hover{
        background-color: rgb(255, 43, 90);
        border-color: White;
        
        filter:alpha(opacity=50);
        -moz-opacity:0.5;
        -khtml-opacity: 0.5;
        opacity: 0.5;
      }
Default the instance that your Login shows
When the Login is shown without any page triggering it, you can default the instance on the dropdown via the tm1-default-instance.
This only applies if there is more than one instance on your application (i.e. dev and uat).
